# Guia de diseno

## Objetivo visual

El portafolio debe verse profesional, personal y actual. Debe evitar verse como una plantilla generica sin identidad.

## Principios visuales

- claridad primero
- jerarquia tipografica fuerte
- secciones bien diferenciadas
- uso intencional del color
- buen equilibrio entre personalidad y legibilidad

## Tipografia

Se recomienda combinar una tipografia principal legible con una tipografia de apoyo que aporte identidad. La seleccion final debe implementarse con `next/font`.

## Color

La paleta debe definir:

- color base de fondo
- color principal de acento
- color secundario
- escala neutra para texto y superficies

Debe evitarse una interfaz plana sin direccion visual.

## Layout

La pagina principal debe priorizar:

- hero claro
- seccion sobre mi
- proyectos destacados
- habilidades o stack
- contacto final

## Responsive

El diseño debe pensarse desde movil y escritorio. Cada bloque tiene que conservar legibilidad y buena separacion visual en ambas vistas.

## Accesibilidad base

- contraste adecuado
- foco visible
- semantica correcta
- textos alternativos en imagenes relevantes

## Regla general

Cada decision visual debe reforzar la identidad profesional del desarrollador.
