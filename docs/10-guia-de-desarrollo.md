# Guia de desarrollo

## Objetivo

Dejar instrucciones claras para que cualquier persona pueda entender como trabajar sobre este proyecto.

## Flujo esperado

1. instalar dependencias con `npm install`
2. ejecutar entorno local con `npm run dev`
3. desarrollar por secciones segun el plan
4. validar con `npm run lint` y `npm run build`
5. revisar responsive, SEO y rendimiento

## Convenciones recomendadas

- nombres de archivo descriptivos
- componentes pequenos y reutilizables
- contenido desacoplado de la UI cuando sea posible
- documentar decisiones importantes

## Como agregar un proyecto nuevo

1. crear o actualizar su contenido fuente
2. agregar thumbnail o recursos necesarios
3. mostrarlo en el listado principal si corresponde
4. crear detalle individual cuando exista esa ruta

## Como mantener la documentacion

Cada cambio relevante en alcance, estructura o arquitectura debe reflejarse en `docs/`.
